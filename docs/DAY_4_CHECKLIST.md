**Why Document Before Designing?**
- Ensures consistency across all screens
- Provides exact specifications for implementation
- Makes Figma design faster (no guessing)
- Creates single source of truth
- Enables parallel work (design + development)

**Color System**
- Primary Blue: Trust, reliability (campus platform)
- Secondary Green: Success, found items
- Semantic colors: Clear status communication
- Gray scale: Professional, readable
- Category colors: Quick visual identification

**Typography**
- System fonts: Fast loading, native feel
- Clear hierarchy: Display → H1 → H2 → Body
- Readable sizes: 16px base for body text
- Proper line heights: 1.5 for readability

**Components**
- Reusable: Build once, use everywhere
- Variants: Different states (hover, active, disabled)
- Auto Layout: Responsive by default
- Consistent: Same patterns across app
